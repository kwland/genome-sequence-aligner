    How it works

    Course Hybrid AI (default, 20,000-particle ceiling). This mirrors the Python tournament submission on the official 8×12 board and 5/4/3/2/2 fleet. It maintains complete legal fleet hypotheses, filters them using only HIT/MISS/UNKNOWN board state, scores occupancy and partially hit ships, and applies a shallow two-shot lookahead. It intentionally ignores the website's sunk-ship announcement because the official starter does not provide one. The playable opponent uses the full particle ceiling; the 100–200-game benchmark uses a smaller, clearly labeled population so it finishes in a reasonable time.

    POMCP AI (research planner). This is a strong practical approximation, not a proof of optimal play. It treats Battleship as a partially observable planning problem. It maintains a particle belief over complete legal fleets, samples a possible hidden fleet for each simulation, then uses an online PUCT/Monte-Carlo tree search over future shot and observation branches. Unlike a purely greedy heatmap, it can prefer a shot because the hit/miss result improves later choices. Exact sunk-ship announcements are included as legal observations.

    Bayesian AI. Maintains a persistent population of thousands of "particles" — complete, internally consistent guesses at the whole fleet's layout — across the entire game, filtering (not rebuilding) it after every shot. A particle survives a miss only if none of its ships are there, and survives a hit only if one of its ships is. Cells are scored from what fraction of surviving particles agree on them, split into two questions: is anything here at all (occupancy), and does a ship we've already found but not finished continue into here (active-ship / sink probability). Once any hits are on the board, the second question dominates the score, so it finishes ships it's found before hunting new ones. Retaining each particle's individual ship identities — instead of collapsing a particle into a single "occupied cells" set — is what lets it reason correctly when two ships happen to be placed touching each other: it never has to decide which specific ship a run of hits belongs to, since every consistent interpretation stays alive in the population until later shots naturally rule the wrong ones out.

    Probability AI (heuristic). A much faster approximation: for every ship length in the fleet, it enumerates every placement still consistent with the board and tallies how many valid placements cover each cell, independently per ship length. A placement only counts toward "finishing a found ship" if it still has an unknown cell to give (not if it's already fully revealed) — good enough to avoid wasting shots on a fully-explained run, but unlike the Bayesian AI it scores each ship length in isolation, so it can't fully rule out an option that looks locally plausible but is actually impossible once the rest of the board is taken into account.

    Ship placement. A single fixed board is never the best repeat-game defense because it can be memorized. The default placement policy therefore samples from a weighted mixed strategy generated offline. The optimizer evolves thousands of legal layouts, evaluates finalists against seeded POMCP, Bayesian, Probability, and human-style hunt/target policies, and solves a finite zero-sum maximin linear program. It also constrains the pool's aggregate cell occupancy so no small group of coordinates becomes an obvious opening target. Between games, the browser may softly tilt those maximin weights away from cells the player tends to fire at early, while retaining random exploration and avoiding recently used layouts. The placement benchmark compares this strategy directly with random and uniformly sampled elite fleets.
